# Website Architecture Description

This is a static academic portfolio website built on **Hugo 0.136.5** using the **Hugo Blox Builder** framework (formerly Hugo Academic), which employs a modular component-based architecture with Hugo modules for theme management. The site uses **Tailwind CSS** (via the blox-tailwind module) for styling and includes a **Netlify plugin** for optimized deployments. Content is organized in Markdown files within structured directories (`content/authors/`, `content/publication/`, `content/post/`, `content/project/`, `content/event/`, `content/teaching/`) with YAML front matter, while site configuration is managed through YAML files in `config/_default/` (hugo.yaml for core settings, params.yaml for theme parameters, module.yaml for Hugo module imports, and menus.yaml for navigation). The architecture includes a custom Python sync script (`sync_personal_info.py`) that maintains a single source of truth (`PERSONAL_INFO.md`) and automatically propagates personal information across multiple config files. The site generates static HTML output to a `public/` directory, is configured for GitHub Pages deployment (with Netlify as an alternative), supports dark/light theme switching, includes search functionality via Pagefind, and uses Hugo's built-in taxonomies for authors, tags, and publication types. Custom layouts can be overridden in the `layouts/partials/` directory, and the modular blox system allows for component-based page building with community and all-access blox components mounted from the `hugo-blox/blox/` directory.
